# Port-convention source registry

A durable, machine-parseable index of **where each board's canonical port-naming
data lives** — the vendor/community constraint or pin-assignment file (QSF, XDC,
PCF, LPF, CST, …) that defines the top-level HDL port names (clock, LEDs,
buttons, switches, 7-segment) a board-native design would use.

The registry stores **ranked pointers + verification evidence**, not copies of
the files. It exists so that:

- `port_conventions` blocks added to `boards/*/*.json` (consumed by **U21**,
  board-native VHDL mode) always trace back to an authoritative source;
- when a better source appears later, one registry row changes and the
  convention data can be regenerated;
- "we looked and found nothing" is recorded durably (`status = "none-found"`),
  so the search is not silently repeated.

The simulator does **not** read these files at runtime; they are maintainer
data for the (planned) `scripts/sync_port_conventions.py` pipeline and human
reference. See `docs/improvement_roadmap.md` (U21, Icebox P2/P5).

## Files

One TOML per board family: `terasic.toml`, `ice40-hobbyist.toml`,
`digilent.toml` (auto-generated — see below), with more families landing as
their research batches complete.

## Schema

```toml
[[board]]                 # one entry per PHYSICAL board
name = "DE10 Lite"        # display name (matches the board JSON's name)
files = ["amaranth-boards/de10_lite.json"]   # every boards/ JSON this applies to
maker = "Terasic"
status = "verified"       # see below
notes = "history + caveats; for none-found, the queries tried"

  [[board.source]]        # ranked; rank 1 = most authoritative
  rank = 1
  kind = "community"      # vendor-official | official-repo | community | personal | vendor-manual
  format = "QSF"          # QSF | SDC | TCL | XDC | UCF | PCF | LPF | PDC | CST | CCF | ISF | XML | ACF | CSV | PDF | HTML | MD | REPO | OTHER
  url = "https://raw.githubusercontent.com/..."
  fetched = true          # true only if the content was actually retrieved and inspected
  retrieved = "2026-07-11"
  license = "MIT"         # if visible, else "unknown"
  device_evidence = "set_global_assignment -name DEVICE 10M50DAF484C7G"
  clk = "MAX10_CLK1_50"   # extracted port names (advisory — parsers re-extract at
  leds = "LEDR[9..0]"     # generation time; the load-bearing columns are
  buttons = "KEY[1..0]"   # identity, url, rank, fetched)
  switches = "SW[9..0]"
  seven_seg = "HEX0..HEX5, 7 bits per digit"
  active_low = "not visible in this source"   # only with evidence; polarity usually needs the manual
  evidence = ["set_location_assignment PIN_A8 -to LEDR[0]"]
```

**Status semantics:** `verified` = at least one fetched source plausibly for
this exact board · `candidate` = a real lead (gated download, repo) not
fetch-verified · `none-found` = no constraint-file lead located (queries listed
in notes) · `not-attempted` = not yet researched.

**Source kinds:** `vendor-official` (the board maker / FPGA vendor itself) >
`official-repo` (the board's own project org) > `community` (well-known
collections, course material, reverse-engineering canon) > `personal`
(individual mirrors). `vendor-manual` marks documentation that authoritatively
answers polarity/pin questions without being a constraint file.

## How it is built and maintained

Discovery is batched per maker family: small research agents follow
a fetch-and-quote rubric (every recorded URL fetched; device line quoted when
present; clock extraction mandatory; no invented paths; honest `none-found`),
and results are normalized, adjudicated, and spot-verified before landing here.
Facts added during verification carry a "verified by direct fetch/GitHub API
<date>" stamp in the row.

`digilent.toml` is different: those boards' `port_conventions` are already
mechanically generated by `scripts/sync_digilent_xdc.py` from Digilent's
master-XDC repo (MIT), so its rows are auto-generated from each board JSON's
`source` block, with commit-pinned raw URLs.

To improve an entry: add/replace the source row (keep ranks consecutive,
`fetched` honest), note the change, and — once the conventions pipeline exists —
re-run it against the new source. Never delete an outranked source that is
still live; demote it.
