### U4. Error messages with contextual hints ✅

- ✅ **2026-07-07 (PR #181, issue #173).** Stage-2 `check_vhdl_contract()` now **parses the toplevel entity's port/generic clauses** (comments stripped, balanced-paren clause extraction, scoped to the entity matching the filename — so the multi-entity embedded-core files are never mis-inspected; an unparseable interface falls back to the legacy whole-text scan, keeping exotic formatting permissive) and reports precise, board-aware errors: **fixed-literal port widths vs the selected board's resource counts** (the flagship *"Port 'led' is a fixed 16 bits wide, but DE10-Lite has 10 LEDs. Stage-3 failures pass through a new **`add_error_hints()`**: four rule families matched against *empirically captured* GHDL and NVC wordings (missing IEEE header, unmapped generic, unconnected extra port, vector-length mismatch — the last quoting the board's real `NUM_*` values and naming the failing port parsed from the echoed wrapper association, which covers expression-width bugs like the committed `9 * NUM_SEGS` fixture that width regexes can't see); unrecognized stderr passes through unchanged. `ErrorDialog` gains `example_path` → a **[View Example]** button (+ `V` key, `btn_load_vhdl` style so all three themes are covered) that opens the board-appropriate example (`counter_7seg.vhd` on 7-seg boards, else `blinky.vhd`; chosen by `controller.example_vhdl_for()`) with the platform opener and keeps the dialog up for side-by-side comparison; wired at all five validation dialog sites. Two new picker-facing fixtures (`bad_contract_fixed_width.vhdl`, `bad_contract_wrong_direction.vhdl`); 55 new tests (parsed-contract matrix incl. an embedded-core no-false-positive sweep and comment-mention immunity, canned two-backend stderr hint units, slow hinted-analysis integration on GHDL + NVC, and `tests/test_error_dialog.py` for layout/clicks/keys/opener); all 23 committed designs verified to still pass the contract under plain / 7-seg / no-board contexts, and the dialog verified visually headless at two window sizes. ### U21. Board-native VHDL mode (port conventions) ✅

- ✅ **2026-07-13, released in v0.14.0 — a multi-phase arc (Part A: conventions population; Part B: matcher + native wrapper).** A design written to a board's *own* port names and fixed widths — Terasic `CLOCK_50` / `KEY(3 downto 0)` / `LEDR(9 downto 0)` / `SW(9 downto 0)` / `HEX0`-`HEX5`, with **no `NUM_*` generics** — now simulates unmodified, closing the "Done when" (a DE10-Standard-style native file runs without modification).

  **Groundwork** (#198 registry, #199 arc plan): `docs/port_convention_sources/` holds ranked, fetch-verified pointers to every board's canonical constraint file, and `docs/u21_board_native_vhdl_plan.md` is the phase-by-phase execution plan (with per-phase verification and quality gates).

  **Part A — populate `port_conventions` across the fleet.** **A0** (#209, issue #200) schema deltas: `seg_port_mapping.style` gains `per_segment_scalars` / `scan` / `serial`; a typed `digit_enable` strobe for `scan`; a `source` provenance stamp + `naming` (`canonical`/`project-derived`) flag; `leds_green` promoted to a typed `port_mapping`. **A1** (#210, issue #201) a re-sync preservation guard in `scripts/sync_common.py`'s `write_outputs()` so an upstream re-sync can never silently wipe hand-authored / populated conventions (per-top-level-key merge; the *merged* result is what gets schema-validated). **A2** (#211, issue #202) `scripts/port_convention_parsers/` — one pure `parse(text) -> PortTable` per constraint dialect (QSF / XDC / UCF / PCF / LPF / CST / CCF / BoardStore XML) plus a dialect-agnostic `classify()` that buckets ports into a `port_convention`-shaped dict by name-shape alone (incl. three 7-seg structures). **A3** (#212, issue #203) a `port_conventions` generator + curated overlay driven from the registry. **A4** (#215, issue #204) the population waves (first Terasic boards); follow-ups rescued DE10-Lite via a cited overlay name-override (#218) and DE0 with a split-DP 7-seg classifier (#219), and #213 / #214 closed the A2 schema-symmetry and generator row-gate gaps.

  **Part B — consume the data.** **B1** (#216, issue #205) threaded `port_conventions` into `BoardDef` with `to_json` / `from_json` round-trip. **B2** (#217, issue #206) the matcher: `check_vhdl_contract()` returns a frozen `ContractResult(ok, message, match)`, and `match_convention(ports, generics, board_def)` yields a frozen `ConventionMatch` (maker + per-role `NativePort` / `NativeSeg` carrying names / width / `active_low`) — but it held native files at `ok=False`, because the *generic* wrapper can't instantiate a native design (its port map is `clk/sw/btn/led/seg`). **B3a** (#220, issue #207) shipped the missing half: `_render_native_wrapper()` emits a native `sim_wrapper` that instantiates the design by its native names and adapts them to the `clk/sw/btn/led/seg` boundary — inverting active-low LEDs (`led <= not led_uut`) and buttons (`key_uut <= not btn`), packing an `individual` 7-seg per digit (decimal point off) — carrying the *same* entity / generics / top-ports / clock process as the generic wrapper (so `launch_simulation` / `run_cmd` and the cocotb testbench are untouched) but baking the board widths as generic *defaults* so the analysis-time `{}` elaboration lines up with the native fixed widths. The `ok=True` flip lets native designs flow through the existing controller gate; the `ConventionMatch` threads `SessionState.convention` → `analyze_vhdl` / `launch_simulation` → the wrapper, and a native run exports `FPGA_SIM_NATIVE_CONVENTION`. Three `hdl/native/` examples — DE10-Standard (active-**high** `LEDR`), DE0 (split-DP `HEXn_D` + green `LEDG` + `BUTTON`), DE25-Standard (active-**low** `LEDR`) — analyze end-to-end under **GHDL and NVC**; a standalone NVC run proves the inversion (`led == not ledr`). The auto-`.gtkw` for a native run preselects the design's own `sim_wrapper.uut.<native>` signals (+ the board-logical `led` / `seg` so the inversion is visible) instead of the contract names. **B3b** (#221, issue #207) the run affordances: the top-left info strip tags the mode right after the filename — `(native: terasic)` in an accent color (a restrained "pop") vs `(generic)` — mirrored in the window title; **S** (stats) shows a compact `board-native · active-low: LED, KEY, HEX` note (a single small-font line below the existing rows, so the fixed-height panel doesn't grow and the board keeps its size); the spinner reads "Analyzing board-native …"; the session log gains `mode` (`generic`/`native`) and `convention` (maker slug). **B4** (issue #208) docs + closeout (this entry, the CLAUDE.md board-native subsection, roadmap stub, and CHANGELOG).

  **Cross-board safety (the contract).** The simulator always models the *selected* board and the board's convention supplies polarity, so loading a file "meant for" another board is user error that resolves one of two safe ways: a differing port name (e.g. DE10-Standard's `CLOCK_50` vs DE25's `CLOCK0_50`) makes it a **near-miss** — rejected with the mismatch named, never silently coerced or polarity-flipped — or it matches an electrically identical board. The only cross-board full match in the current data is DE23-Lite ↔ DE25-Standard (same names / widths / polarity). A **data-invariant regression test** sweeps every canonical-convention board, cross-matches all pairs, and asserts no cross-board full match ever diverges on polarity — so it **fails loudly** if a future board introduces a name+width-identical, polarity-different collision, forcing a conscious discriminator. Global cross-board *ambiguity* detection (Icebox **P15**) and Surfer signal preselection via `-c`/`--command-file` (Icebox **P16**) are parked as orthogonal to U21.

  **Scope / tests.** Only the `individual` 7-seg style is adapted; `scan` / `serial` / `per_segment_scalars` / `packed_vector` stay generic → **U22**. New `tests/test_native_convention.py` covers the wrapper-gen invert path (per shape), the `.gtkw` native-vs-generic preselection, the cross-board polarity invariant, the DE10-on-DE25 near-miss, GHDL + NVC e2e for all three examples, the DE25 standalone inversion, and the B3b subprocess helpers; the B2 matcher / session-log / sim-panel suites were updated. ### U31. Board-native partial-interface support ✅

- ✅ **2026-07-13, released in v0.14.0 (PR #226, issue #223).** Relaxed the board-native matcher so a design need only declare the roles its board's convention names, not the full clk + LEDs + switches + buttons set. `_attempt_convention` now requires **clk + LEDs** as the minimum and matches **switches / buttons only when the convention declares them** — generalizing the existing "7-seg required only when the board physically has a display" rule — so a design for a **switch-less or button-less** board (most FPGA boards have no switches) simulates unmodified. The requirement keys off the *convention* (the native-name source of truth), not the board's physical resources, so a board whose convention hasn't captured its switches yet simply can't drive them natively rather than never matching. `ConventionMatch.switches` / `.buttons` became `NativePort | None`, rippling through every consumer (`_native_convention_message`, `_native_gtkw_signals`, the `FPGA_SIM_NATIVE_CONVENTION` env export, and the cross-board polarity invariant test). `_render_native_wrapper` **ties off** an absent input bank — the wrapper's top `sw` / `btn` boundary ports stay (cocotb still reads `dut.sw` / `dut.btn`) but are floored to a one-bit dummy and left unconnected to the design, the exact `NUM_* = max(1, count)` floor the generic path already uses in `controller.build_generics` — while an absent output bank is left dark. **Honest near-miss:** a design declaring an *input* the convention lacks stays a near-miss (it would otherwise be an unbound uut port — an elaboration error), whereas an unmapped *output* is legally left `open`, exactly as the DE0 example's split-DP `HEXn_DP` pins already are. On today's full-interface conventions behavior is unchanged — the value is as the multiplier for **U32** (litex/amaranth auto-derived conventions): partial-interface support lifts the framework-derived ceiling from ~52 boards to ~238. Verified end-to-end by running a genuinely switch-less synthetic board (`10 LEDs · 0 BTN · 0 SW`) with an LED-only native design through the full headless cocotb + `sim_testbench` loop (`interactive_sim PASS`), plus GHDL + NVC analysis of the tied-off wrapper. New tests in `tests/test_native_convention.py` cover the LED-only / LED+btn / LED+sw matches, the extra-*input* near-miss vs. extra-*output*-allowed distinction, the wrapper tie-off, the None-safe `.gtkw` preselection, and GHDL/NVC e2e of a partial wrapper. ### U32. Auto-derive `port_conventions` from the litex & amaranth platform files ✅

- ✅ **2026-07-14, released in v0.14.0 (bundled with the U21 + U31 board-native release).** The litex and amaranth sync parsers now emit a framework-derived `port_conventions.{litex,amaranth}` block for **241/246** boards (164/167 litex + 77/79 amaranth; the five hold-outs lack a clock or LEDs — the clk + LEDs floor U31 established). Verified end-to-end: a committed litex Arty design (`hdl/native/arty_litex.vhd`, LiteX names, no `NUM_*`) simulates unmodified through the full headless cocotb + `sim_testbench` loop on **both GHDL and NVC** (`interactive_sim PASS`). ### U34. Single-window simulation ✅

- ✅ **2026-07-17, released in v0.15.0.** The launcher's pygame window now persists for the **entire session** — simulation no longer closes it and opens a second window. The child (`sim/sim_testbench.py`, now pygame-free) drains `input`/`speed`/`clk`/`pause`/`stop` and sends throttled `state` + a final `bye`; the host never blocks on it (all `poll(0)` drains, startup awaited with a spinner + 90 s watchdog, shutdown = stop-message → bounded wait → terminate). **The VHDL boundary is untouched** — the child reads the same `sim_wrapper` ports for generic and board-native runs alike, so U21/U31/U32/U33 needed nothing. **Why:** the window flip was never a performance feature (draw is ~1.3 % of frame time); the UI lived in the sim process only because cocotb must run there. - **Benchmark** became two modes: default `--benchmark N` drives the real `SimulationScreen` headless (SDL dummy driver, 60 fps, toolbar hidden) so the number still covers the whole app, plus a new `--benchmark N --no-ui` that runs the child alone (no pygame) to isolate simulator throughput from UI cost — the two double as a host/child CPU-contention probe. **Stats panel** G/D/I denominators changed (G = the child's own loop share, D/I = host-frame shares, so they need not total 100 %), and the child streams sim-time totals in throttled bursts, so `SimPanel.set_remote` measures throughput against wall time between advancing samples (fixing a Clk/frame ping-pong). **U24** (batch `Timer` calls) was closed *won't-do* by this arc's spike (×10 batching gained only +3 %). Icebox parks **P19** (`analyze_metrics.py` split-process rework) and **P20** (modernize `capture_frames.py`).
