### U4. Error messages with contextual hints ✅

- ✅ **2026-07-07 (PR #181, issue #173).** Stage-2 `check_vhdl_contract()` now **parses the toplevel entity's port/generic clauses** (comments stripped, balanced-paren clause extraction, scoped to the entity matching the filename — so the multi-entity embedded-core files are never mis-inspected; an unparseable interface falls back to the legacy whole-text scan, keeping exotic formatting permissive) and reports precise, board-aware errors: **fixed-literal port widths vs the selected board's resource counts** (the flagship *"Port 'led' is a fixed 16 bits wide, but DE10-Lite has 10 LEDs. The simulator sets NUM_LEDS=10 for this board — declare the port as `led : out std_logic_vector(NUM_LEDS - 1 downto 0)`"*; a fixed width equal to both the board count *and* the wrapper default still passes, since that combination genuinely works end-to-end), **wrong port modes** (GHDL *and* NVC silently accept `led : in` — the sim would run with permanently-undefined LEDs, so the textual check is the only guard, verified empirically), **missing required generics** upgraded from a console-only `[warn] … will use VHDL defaults` (factually wrong — the wrapper maps all four unconditionally, so these always failed later with a cryptic `sim_wrapper.vhd` error) to a fatal error showing the exact generic block, **extra input ports / generics without defaults**, and **`seg` without `NUM_SEGS`** on 7-seg boards. Stage-3 failures pass through a new **`add_error_hints()`**: four rule families matched against *empirically captured* GHDL and NVC wordings (missing IEEE header, unmapped generic, unconnected extra port, vector-length mismatch — the last quoting the board's real `NUM_*` values and naming the failing port parsed from the echoed wrapper association, which covers expression-width bugs like the committed `9 * NUM_SEGS` fixture that width regexes can't see); unrecognized stderr passes through unchanged. `ErrorDialog` gains `example_path` → a **[View Example]** button (+ `V` key, `btn_load_vhdl` style so all three themes are covered) that opens the board-appropriate example (`counter_7seg.vhd` on 7-seg boards, else `blinky.vhd`; chosen by `controller.example_vhdl_for()`) with the platform opener and keeps the dialog up for side-by-side comparison; wired at all five validation dialog sites. Two new picker-facing fixtures (`bad_contract_fixed_width.vhdl`, `bad_contract_wrong_direction.vhdl`); 55 new tests (parsed-contract matrix incl. an embedded-core no-false-positive sweep and comment-mention immunity, canned two-backend stderr hint units, slow hinted-analysis integration on GHDL + NVC, and `tests/test_error_dialog.py` for layout/clicks/keys/opener); all 23 committed designs verified to still pass the contract under plain / 7-seg / no-board contexts, and the dialog verified visually headless at two window sizes. Carried-forward rule noted in the [Delivery log](#delivery-log).

## Delivery log

Cross-cutting notes carried forward from completed work (forward-relevant gotchas also appear as ⚠ notes on the open cards they affect in [improvement_roadmap.md](improvement_roadmap.md)):

- **Contract checks are parse-based; hint wordings are pinned to real simulator output (from U4 ✅).** New contract rules belong in `sim_bridge._check_parsed_contract()` (operating on `_IfaceDecl`s), not new whole-text regexes — and must keep the unparseable-interface fallback permissive, since stage 3 + `add_error_hints()` still catches what the parser can't see. The `add_error_hints()` rule regexes match the *observed* GHDL 6/7 and NVC 1.2x error wordings — when bumping either simulator, re-run the broken-design sweep (fixed width, missing generic, extra port, missing IEEE header, on both backends) and adjust the patterns if the wording drifted; a non-matching message silently loses its hint (tests cover the current wordings).
- **Selector key handling (from U1 ✅).** Any new *printable* keyboard shortcut on the board selector must be intercepted in `BoardSelector._handle_keydown()` *above* the `filter_text += ev.unicode` branch (match on `ev.unicode`), or the keystroke leaks into the text filter — this is how `?` and type-to-filter coexist.
- **Off-main-thread work (from U2 ✅).** `run_with_spinner()` is the pattern for any future long-running launcher operation (simulator install probe, board re-sync, large-file load): the worker callable runs on a `ThreadPoolExecutor` thread and **must not touch pygame** — only the main thread draws. Pass arguments with `functools.partial` (not a `lambda`) when the call site is inside a loop, so loop variables are bound eagerly (avoids B023 and the closure-widening that would otherwise re-introduce `str | None`).
- **Session file is merge-on-write (from U5 ✅).** Any new persisted key goes through `update_session()` / `save_session()` (both merge into the existing JSON) — never rewrite the file wholesale, and never write a key another writer owns: `speed_factor` belongs to the **sim subprocess** (the launcher only reads it, freshly, before each launch), the launcher owns the board/VHDL/prefs/window keys, and the Settings dialog owns `theme` (+ `metrics_enabled` / `waveform_enabled` when U19/U10 land). Settings-dialog rows apply immediately (no OK/Cancel) — keep that pattern for new rows. Tests that construct a controller or dialog must redirect `fpga_sim.session_config.SESSION_FILE` (see the autouse fixtures in `test_controller.py` / `test_settings_dialog.py`) or they will write the developer's real `~/.fpga_simulator/session.json`.
- **Sim→launcher signalling is an exit-intent sidecar file, trusted only on a clean exit (from U7 ✅).** The simulation subprocess reports *why* it ended by writing a `SimExit` value to the file named in `FPGA_SIM_EXIT_INTENT_FILE`; `launch_simulation()` clears any stale copy first (the reload path reuses the work dir) and honors the file **only when the process returned 0**, so a GHDL/NVC crash is never read as navigation. A new sim-screen action (e.g. an in-sim [Save waveform] under U10) adds a `SimExit` member + a toolbar button (borrow a `ButtonStyle` role by *name* in `ui/sim_toolbar.py` so themes apply automatically) and a routing arm in `ScreenController.on_simulate()` — do **not** overload the process return code, which the simulators own. To drive the real `sim_testbench` loop in a test/verify harness without a human, inject a scripted-events module ahead of `sim_testbench` in `COCOTB_TEST_MODULES` (it patches `pygame.event.get`); the loop, toolbar, and intent-file write then run for real (recipe in `memory/reference_headless_sim_testing.md`).
